- 2026-06-07: `set_alarm` payloads should include `label` as an empty string when no label is provided to mirror the `clock-server` MQTT payload behavior.
- 2026-06-07: Read-side device state and event visibility should ideally be in v1, scoped to the device topics defined by `clock-server/docs/lcd-reference.md`.
- 2026-06-07: Lightweight reviewer pass completed; read-side MCP tools were made explicit as `get_device_state`, `get_recent_events`, and `get_command_results`.
